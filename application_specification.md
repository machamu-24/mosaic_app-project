# アプリケーション仕様書 (モザイク処理アプリ)

## 1. 課題と解決策

### 課題
* **プライバシー保護の必須要件**: 医療現場（TUGテストやリハビリの歩行分析など）や研究用途において、動画および写真データを記録・活用する際、被写体や背景に映り込んだ人物のプライバシー保護（個人情報の匿名化）が不可欠です。
* **手作業の限界**: 動画に対して手作業でモザイク処理を行うのは、莫大な時間と労力がかかるため現実的ではありません。
* **既存モデルの精度不足**: 一般的な顔検出技術では、「マスクを着用している顔」「遠く離れた小さな顔」「横顔」や「部分的にしか見えない顔」の検出精度が低く、検出漏れ（モザイクの掛け忘れ）が発生するリスクがありました。

### 解決策
* **高精度AIの導入**: 最新かつ高性能な物体検出モデル「YOLOv8」を顔検出に特化させて学習させたモデル（YOLOv8-Face）を採用することで、悪条件下でも高精度に顔を検出し、自動でモザイク処理を適用するスクリプトを構築し、課題を解決します。

---

## 2. 目的と実装

### 目的
動画および写真内のすべての顔に対して自動的かつ確実にモザイク処理を施し、個人を特定できない安全なファイル（匿名化データ）を効率的に生成すること。最終的には、非エンジニアのユーザーでも手軽に利用できるGUIアプリケーションなどへの展開を見据えた基盤となります。

### 最低限の実装内容（現在の機能）
* **コマンドラインインターフェース (CLI)**
  * スクリプト実行時に引数で入力動画と出力動画のパスを指定し、自動処理を行います。
* **モデルの自動ダウンロード機能**
  * 初回実行時や環境構築時に、必要なYOLOv8-Faceの重みファイル（`.pt`）が存在しない場合、GitHubから自動的にダウンロードしてロードします。
* **単一画像およびフレーム単位の検出とモザイク適用**
  * 入力ファイルの拡張子に基づいて、動画か静止画（写真等）かを文脈から自動で判別します。
  * `OpenCV` を用いて動画をフレームごとに、または画像ファイルをそのまま読み込み、YOLOの推論結果から顔のバウンディングボックス（矩形領域）を特定。
  * 該当領域を意図的に低解像度にリサイズ（ピクセル化）することでモザイク効果を施し、新たなファイルとして書き出します。
* **柔軟なパラメータ調整**
  * モザイクの強度（画素ブロックのサイズ）、YOLOの推論解像度（`imgsz`：デフォルト 960）、信頼度閾値（`conf`：デフォルト 0.25）、IoU閾値（`iou`：デフォルト 0.45）を必要に応じてコマンドライン引数で調整可能です。
* **デバイスの最適化とフォールバック**
  * MacのApple Silicon (M1/M2等) 環境において、MPS (Metal Performance Shaders) を用いた高速化を試みます。（※一部未サポートの演算に対しては自動でCPUフォールバックを行うことでエラーを回避します）

## 3. 技術スタック

* **プログラミング言語**: Python 3.9 以上
* **主要ライブラリ・バージョン**:
  * `ultralytics == 8.3.17`: YOLOv8モデルの読み込み、推論実行の中核を担う公式フレームワーク。
  * `opencv-python == 4.10.0.84`: 動画ファイルの読み書き、フレーム画像の操作、およびモザイク（リサイズ）処理を担当。
  * `numpy == 1.26.4`: 行列計算等の数値処理用。（※PyTorchやUltralyticsとの互換性の問題を回避するため、安定系の2.0未満であるv1.26.4を指定しています）

### 使用しているYOLOの詳細 (YOLOv8-Face)

本アプリケーションの顔検出エンジンには、Ultralytics社製の「YOLOv8」を採用し、開発コミュニティによって「顔検出用」にファインチューニングされた事前学習済みモデル（*lindevs/yolov8-face* 等）を使用しています。

1. **アーキテクチャの優位性**
   YOLO（You Only Look Once）は元々リアルタイム性能と精度のバランスに優れたモデルですが、v8ではアンカーフリー（Anchor-free）設計が採用され、より柔軟かつ高精度なバウンディングボックスの予測が可能になっています。
2. **顔検出への特化**
   専用の大規模な顔画像データセットを用いて学習されており、マスク装着顔、メガネ・サングラス装着顔、横顔、あるいは画面の奥にいる小さな顔に対しても、非常に高いRecall（見逃しの少なさ）を誇ります。これにより、医療情報保護の要件に耐えうる「モザイク漏れの防止」に寄与しています。
3. **モデルサイズと推論解像度のチューニング**
   * **モデルサイズ**: 用途と処理速度に合わせて複数のサイズ（n, s, m, l, x）があり、本スクリプトでは実行速度と精度のバランスに優れた **Mediumモデル (`yolov8m-face`)** をデフォルトとして採用しています。
   * **高解像度推論 (`imgsz=960`)**: 小さな顔を正確に捉えるために、デフォルトの推論画像サイズを一般的な640から「960」に引き上げています。これにより遠距離の顔の検出率を高めています。
